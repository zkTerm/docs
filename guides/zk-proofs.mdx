## Overview

zkTerm implements multiple ZK proof systems for different use cases:

- **Groth16**: File encryption password verification
- **STARK**: Identity verification on Starknet
- **Ed25519**: Digital signatures for zkID

## Groth16 Proofs

Used for zkStorage password verification.

### Generate Proof

```bash
zk proof generate --inputs "a=5,b=7"
```

### How It Works

```
Password → Poseidon Hash → ZK Circuit → Groth16 Proof
```

The proof confirms you know the password without revealing it.

### Implementation

```typescript
import { generatePasswordProof } from '@zkterm/zkstorage';

const proof = await generatePasswordProof({
  password: 'mypassword',
  expectedHash: '0x123...'
});

// Proof contains: pi_a, pi_b, pi_c, publicSignals
```

## STARK Proofs

Used for zkID on-chain verification on Starknet.

### Generate STARK

```bash
zkid verify
```

This creates a STARK proof and submits it to Starknet.

### Cairo Contract

```cairo
#[starknet::contract]
mod ZkIDProofRegistry {

```

## Proof Verification

### Client-Side

```typescript
import { verifyProof } from '@zkterm/zkstorage';

const isValid = await verifyProof(proof, publicSignals);
```

### Server-Side

```bash
curl -X POST https://zkterm.io/api/zk-toolkit/proof/verify \
  -H "Content-Type: application/json" \
  -d '{
    "proof": {...},
    "publicSignals": [...]
  }'
```

## Performance

| Proof System | Generation Time | Verification Time | Proof Size |
|--------------|-----------------|-------------------|------------|
| Groth16 | ~5-10s | ~100ms | ~200 bytes |
| STARK | ~2-5s | ~500ms | ~1KB |
| Ed25519 | ~1ms | ~2ms | 64 bytes |

## Advanced Topics

### Custom Circuits

Create your own ZK circuits using circom:

```circom
template MyCircuit() {
    signal input a;
    signal input b;
    signal output c;
    
    c <== a * b;
}
```
