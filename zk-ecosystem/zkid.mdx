## Overview

zkID is a privacy-preserving universal identity system that generates a `did:zk` decentralized identifier with on-chain verification on both Solana and Starknet.

<Info>
  zkID uses Ed25519 cryptography for proof generation and supports dual-chain verification for maximum security and decentralization.
</Info>

## DID Format

```
did:zk:[12 random alphanumeric characters]
```

**Example:**
```
did:zk:a7Bk9mX2pQ4r
```

## How It Works

<Steps>
  <Step title="Create Identity">
    Generate a new zkID with a username and password. The system creates:
    - Ed25519 keypair for signing
    - Unique 12-character DID
    - Password hash using bcrypt
  </Step>
  <Step title="Generate Proof">
    Client-side proof generation using Ed25519 signatures. The proof demonstrates ownership of the identity without revealing the private key.
  </Step>
  <Step title="On-Chain Verification">
    The proof is verified on-chain via:
    - Solana Mainnet transaction
    - Starknet Sepolia Cairo contract (ZkIDProofRegistry)
  </Step>
</Steps>

## Terminal Commands

### Create zkID

```bash
zkid create
```

Prompts for username and password, then generates your unique `did:zk` identity.

### Verify Identity

```bash
zkid verify
```

Generates and verifies an Ed25519 proof of identity ownership.

### Check Status

```bash
zkid status
```

Shows your zkID and on-chain verification status on both Solana and Starknet.

### View Profile

```bash
zkid profile
```

Displays your complete zkID profile including creation date and verification history.

### Create zkID

## On-Chain Architecture

### Solana Verification

zkID proofs are stored on Solana Mainnet using a Program Derived Address (PDA) structure:

```
PDA = [
  "zkid",
  did_bytes,
  program_id
]
```

### Starknet Verification

The `ZkIDProofRegistry` Cairo contract stores zkID proofs on Starknet Sepolia:

```cairo
#[storage]
struct Storage {
    proofs: LegacyMap<felt252, ZkIDProof>,
    owner: ContractAddress,
}

struct ZkIDProof {
    did_hash: felt252,
    public_key: felt252,
    timestamp: u64,
    verified: bool,
}
```

## Security Considerations

<Warning>
  Never share your zkID private key. The proof generation happens entirely client-side.
</Warning>

**Security Features:**
- Ed25519 elliptic curve cryptography
- bcrypt password hashing (12 rounds)
- Client-side proof generation (private key never leaves device)
- Dual-chain verification for redundancy
- STARK proofs for quantum resistance (Starknet)
