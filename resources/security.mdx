### Encryption Standards

- **Poseidon**: ZK-optimized hash (~300 constraints)
- **SHA-256**: Standard cryptographic hash
- **MiMC**: Alternative ZK-friendly hash

## Best Practices

### For Users

<AccordionGroup>
  <Accordion title="Password Security">
    - Use 16+ character passwords
    - Include numbers, symbols, uppercase
    - Never reuse passwords
    - Use password manager
    - No password recovery - keep safe!
  </Accordion>

  <Accordion title="Key Management">
    - Export private keys to secure location
    - Never share private keys
    - Store backups offline (encrypted USB)
    - Consider hardware wallet for high-value keys
  </Accordion>

  <Accordion title="File Storage">
    - Verify checksums after download
    - Keep backup of important CIDs
    - Use strong, unique passwords per file
    - Don't upload sensitive data without encryption
  </Accordion>

  <Accordion title="Blockchain Operations">
    - Verify transaction details before signing
    - Use hardware wallet for large amounts
    - Check contract addresses carefully
    - Monitor gas prices
  </Accordion>
</AccordionGroup>

### For Developers

<AccordionGroup>
  <Accordion title="API Security">
    ```typescript
    // Always use HTTPS
    const response = await fetch('https://zkterm.io/api/...', {
      credentials: 'include', // For cookies
      headers: {
        'Content-Type': 'application/json'

    // Validate all inputs
    ```

  <Accordion title="Error Handling">
    ```typescript
    // Don't expose sensitive errors
    try {
      await zkStorage.download(cid, password);
    } catch (error) {
      // Bad: console.log(error.stack);
      // Good:
      console.log('Decryption failed');
    }
    ```
  </Accordion>

  <Accordion title="Rate Limiting">
    ```typescript
    // Implement exponential backoff
    const delay = Math.min(1000 * Math.pow(2, attempts), 30000);
    await new Promise(r => setTimeout(r, delay));
    ```
  </Accordion>
</AccordionGroup>

## Smart Contract Security

### Starknet Contract

Our zkID registry contract:

```cairo
#[starknet::contract]
mod ZkIDProofRegistry {
    // Immutable storage
    #[storage]
    struct Storage {
        proofs: LegacyMap<felt252, ProofData>
    }
    
    // Access control
    fn verify_proof(ref self: ContractState, zkid: felt252) {
        // Only proof owner can verify
        assert(caller == proof.owner, 'Unauthorized');
    }
}
```

**Security features**:
- Immutable proof storage
- Owner-only verification
- Gas-optimized operations
- Audited by Starknet Foundation

### Solana Program

Ed25519 signature verification (native):

```rust
// Native Solana instruction
ed25519_verify(
    signature,
    public_key,
    message
);
```
